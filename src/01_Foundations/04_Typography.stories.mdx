## Headings

Headings are typography components used to create various 
levels of hierarchies between text. The `h1` to `h6` elements 
represent the six levels of section and/or page headings with 
`h1` being at the highest hierarchical level and `h6` being at 
the lowest. You should only use one `h1` per page. However, 
using more than one will not result in an error, but using only 
one is seen as best practice. Having a single top-level title is 
also arguably better for screenreader users, and for SEO.

<Canvas className="ally-headings" style={{ margin: "0", }}>
<h1>H1 Heading</h1>
<h2>H2 Heading</h2>
<h3>H3 Heading</h3>
<h4>H4 Heading</h4>
<h5>H5 Heading</h5>
<h6>H6 Heading</h6>
</Canvas>

## Paragraphs

Paragraphs are represented in visual media as blocks of text structured to
group related content and separate them from adjacent blocks
by blank lines. The `p` element is used to create paragraph components.

<Canvas style={{ margin: "0", }}>
<p>This is a paragraph body text example.</p>
</Canvas>


<div id="Links" style={{ marginTop: "80px", }}></div>

## Links

Links allow users to navigate to a different location. They can 
be presented inline inside a paragraph or as standalone text.
The `a` element with its `href` attribute, create hyperlink components
know as an Anchor link which can lead to web pages, files, email addresses,
locations, phone numbers and sms etc in the same page, or anywhere else a URL can
address. Content within each `a` should indicate the link's
destination. Underline should only be used for text links
and should never be used as a mechanism for adding emphasis.
While using this design system you should use the `ExLink`
Atom component for external links, the `NavLink` component
for navigation and the `SkipLink` to skip to main content to
help improve accessibility.

<Canvas style={{ margin: "0", }}>
<p>This is an <a href="#">Anchor link</a> example.</p>
</Canvas>

## Microcopy

Microcopy should be used to represent side comments and
small print, such as copyright and legal text. The `small`
element should be used instead of simply a smaller
font size for it's semantic meaning.

<Canvas style={{ margin: "0", }}>
<p><small>Microcopy text example.</small></p>
</Canvas>

## Emphasis

Emphasis in typography is the strengthening of words in text
with the font of those words in a different style from the
rest of the text. It is the equivalent of prosody stress in
speech. The `em` element is used to highlight emphasis and
can affect the meaning of the sentence itself.

<Canvas style={{ margin: "0", }}>
<p>This is an example of <em>emphasis</em> in text copy.</p>
</Canvas>


<div id="Idiomatic" style={{ marginTop: "80px", }}></div>

## Idiomatic Text

Idiomatic text is a range of text with different semantic
meaning than the surrounding text such as a technical terms,
taxonomical designations, thoughts or words from another
language (which should include the `lang` attribute to
identify the language). The `i` element should be used to
represent this text to help convey meaning when using this
system.

<Canvas style={{ margin: "0", }}>
<p>Examples of <i>idiomatic text</i> could include... 
<br /> "She wondered, <i>What is this writer talking about, anyway?</i>" 
<br /> or <br /> "The Latin phrase <i lang="la">Veni, vidi, vici</i> is often mentioned in music, art, and literature."
</p>
</Canvas>

## Highlighting

Highlight formatting is used for reference or notation purposes. The `mark` element is 
used to represent text which is marked or highlighted. The presence of the `mark` element 
is not announced by most screen reading technology in its default configuration therefore 
for accessibility it should be announced using a span with the `.sr-only` class at the 
start and end of the highlighted text as in the example shown below.

<Canvas style={{ margin: "0", }}>
<p>
<mark>
<span className="sr-only"> [highlight starts] </span> Highlighed Text <span className="sr-only"> [highlight ends] </span>
</mark>
</p>
</Canvas>

## Strikethrough
Strikethrough text renders with a line through it to represent
things that are no longer relevant or no longer accurate. Strikethrough
text is created using the `s` element.
However, this formatting is not appropriate when indicating document 
edits, for edits use the Delete / Insert formatting as below.

<Canvas style={{ margin: "0", }}>
<p>This is an example of <s>Strikethrough</s> text copy.</p>
</Canvas>


<div id="Delete" style={{ marginTop: "80px", }}></div>

## Delete / Insert

The deleted  element represents a range of text that has been deleted 
from a document. This can be used when rendering track changes or source code 
diff information, for example. The inserted element can be used for 
the opposite purpose, to indicate text that has been added to the document. 
The `del` and `ins` elements should be used to create these components.

<Canvas style={{ margin: "0", }}>
<p>This is an example of <del>deleted</del> and <ins>inserted</ins> text copy.</p>
</Canvas>

## Lists

The `ul` element represents an unordered list of items,
rendered as a bulleted list while the `ol` element
represents an ordered list of items renders as a numbered
list. Lists should either be part of a section with other
text for context or have a section heading.

<Canvas style={{ margin: "0", }}>
<ul>
<li>This is an unordered list example</li>
<li>List items that are not in an order</li>
<li>It is just a list of items</li>
</ul>
<ol>
<li>This is an example of an ordered list</li>
<li>Items have a hierarchy</li>
<li>Items are shown with a number</li>
</ol>
</Canvas>

## Figure Captions

The `figcaption` element represents a caption or legend
describing the rest of the content of its parent `figure`
element such as for an image. When using this design system
you should use the `Figure` Atom component for this purpose.

<Canvas style={{ margin: "0", }}>
<figure>
<img src="https://via.placeholder.com/300x200.png?text=Figure+Placeholder" alt="figure placeholder" />
<figcaption>Example figure caption.</figcaption>
</figure>
</Canvas>


<div id="Quotation" style={{ marginTop: "80px", }}></div>

## Inline Quotation

For inline quotation within a paragraph the `q` element should be used to indicate 
that the enclosed text is a short inline quotation. Most modern web browsers implement 
this by surrounding the text in quotation marks and a URL for the source of the quotation 
may be given using the `cite` attribute and as an external link as in the below example.

<Canvas style={{ margin: "0", }}>
<p>When Dave asks HAL to open the pod bay doors, HAL answers: <q cite="https://www.imdb.com/title/tt0062622/quotes/qt0396921"><a title="imdb.com" href="https://www.imdb.com/title/tt0062622/quotes/qt0396921" rel="noopener noreferrer" target="_blank">I'm sorry, Dave. I'm afraid I can't do that.</a></q></p>
</Canvas>


<div id="BlockQuotation" style={{ marginTop: "80px", }}></div>

## Block Quotation

For longer quotations a quotation block component should be used with 
the `blockquote` element which indicates that the enclosed text is an extended quotation.
Usually, this is rendered visually by indentation and a URL for the source of the quotation
may be given using the `cite` attribute, while a text representation of the source can 
be given using the `cite` element. An external link to the cited source may also be given 
if the source is a URL as in the below example.

<Canvas style={{ margin: "0", }}>
<figure>
<blockquote cite="https://www.huxley.net/bnw/four.html">
<p>"Words can be like X-rays, if you use them properly—they’ll go through anything. You read and you’re pierced."</p>
</blockquote>
<figcaption>— Aldous Huxley, <cite><a title="huxley.net/bnw/four" href="https://www.huxley.net/bnw/four.html" rel="noopener noreferrer" target="_blank">Brave New World</a></cite></figcaption>
</figure>
</Canvas>

## Inline Code

Inline code is written inline with other text and is formatted using the `code` element to display its 
contents styled in a fashion intended to indicate that the text is a short fragment of computer code. 
This code content is displayed using the design systems monospace typeface with a darker background 
than other paragraph text.

<Canvas style={{ margin: "0", }}>
<p>The function <code>selectAll()</code> highlights all the text in the input field.</p>
</Canvas>


<div id="BlockCode" style={{ marginTop: "80px", }}></div>

## Code Block

Preformatted text is used to display code within Code Blocks. The `pre` HTML5 element is used to represent the 
preformatted text which is to be presented exactly as written. The text is rendered using the design systems 
monospace typeface and is used wrapped around `code` elements to show the Code Block.

<!-- Note: 
`pre` element does not work within <Canvas> so the example code wraps 
-->
<Canvas style={{ margin: "0", }}>
<pre>
<code>The is a preformatted code block</code>
</pre>
</Canvas>
